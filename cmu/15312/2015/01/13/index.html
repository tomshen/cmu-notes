What does it mean for a programming language to exist?

“Wrong” answers:

  • has a compiler
  • spec in English

A programming language consists of three elements:
1. syntax
2. static semantics: which programs are legal?
3. dynamic semantics (we’re using structured operational semantics in this class): what happens when you execute a legal program?

Inductive judgements over abstract syntax

We can say that n nat means (or rather, we mean for it to mean) that n is a natural number. A judgement is when we choose what syntactic objects mean. We can talk about the n nat statement as a judgement, or for short, the nat judgement. Judgements are not quite the same as propositions. If we have the proposition P, then P true might be a judgement.

Syntactic objects are not strings.

Derivations

Establish a judgement by applying rules until there are no more rules left (reach rules that have no premises). The judgment is the least relation for which the the rules hold. A judgement holds if and only if you can produce a derivation.

The definition of nat is the least relation for which the rules $Z$ and $S$ hold. It is not the case that we want the least number of rules that define nat; we define nat as the smallest predicate that satisfies $Z$ and $S$. Thus, if we want the meaning of $x \texttt{ nat}$ to match our intuition ($x \in \mathbb{N}$), we need to choose rules to produce that meaning.

Structural induction

$a \; is \; b$:

Theorem. If $a \; nat$, then $a \; is \; a$.

Proof. (More pedantic than necessary)
Let $P(n) = n \; is \; n$. We claim that $P$ is closed under the rules for $nat$. closed means what holds for the premise of a rule holds for the conclusion.

We need to show:
1. $P(zero)$
2. If $P(a)$, then $P(succ(a))$.

Case 1:
$zero \; is \; zero$ follows by the rule with no premise.

Case 2:
Suppose $a\;is\;a$. We want to show $succ(a)\;is\;succ(a)$. This follows by the rule with $a\;is\;a$ as its premise.

Theorem. If $\texttt{succ}(a) \texttt{ nat}$, then $a \texttt{ nat}$.

Proof.
By induction, if $b \texttt{ nat}$, then $b \texttt{ nat}$ and $\forall a . b = \texttt{succ}(a) \supset a\texttt{ nat}$.

Case 1:
We want to show $\texttt{zero nat}$ and $\forall a . \texttt{zero} = \texttt{succ}(a) \supset a \texttt{ nat}$. The first part is true by rule $Z$.
Suppose $\texttt{zero} = \texttt{succ}(a)$. However, those are different syntactic objects. (They are fundamentally different things). Thus, this is a contradiction.

Case 2:

Suppose $b \texttt{ nat}$ and $\forall a . b = \texttt{succ}(a) \supset a\texttt{ nat}$ (IH). We want to show $\texttt{succ}(b)$ and $\forall a . \texttt{succ}(b) = \texttt{succ}(a) \supset a\texttt{ nat}$. Let $a$ be arbitrary. Assume $\texttt{succ}(b) = \texttt{succ}(a)$. Then, $b = a$, and thus, $a \texttt{ nat}$.
