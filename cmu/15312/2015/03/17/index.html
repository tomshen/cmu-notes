Aliasing

Two references that govern the same underlying assignable are said to be aliases. The possibility of aliasing complicates reasoning about the correctness of code that uses references, for we must always consider for any two references whether or not they might be aliases.

Free Assignables

In order to be able to have mutable data structures, we need assignables that can live outside the scopes of their declarations. We call these scope-free or free assignables.
