We’re going to discuss abstract process calculus, which is derived from -calculus. The key idea is interacting automata. We will introduce concurrency to Modernized Algol, creating Concurrent Algol, thus applying process calculus to a “real” programming language.

What is concurrency? It’s unrelated to parallelism. Instead, it’s really about composition, and non-determinism (Bob Harper prefers indeterminacy).

Our starting point is consideration of processes as automata. One example is Milner’s vending machine. We have an automata for the vending machine , and an automata for the person using it . They interact concurrently (), since the person loses money as they put it into the machine, and when the machine dispenses a drink, the person gains a drink.

The framing principle is that a process takes the same action regardless of “who’s watching.” Thus, if you have several processors, you can have two of them interact in parallel without affecting the others.

This rule allows for interaction. Thus, the user can get a coffee at the same time the vending machine sends a coffee.
